BLE Audio Credit Window Protocol

          
Live2D Agent 当前使用 GATT 文本命令、PCM 音频写入、状态通知三条 characteristic。音频链路是带背压的生产者-消费者模型。

Audio Packets

            

              
START0x01 + total_len_u32_le + sample_rate_u16_le + channels + format

              
DATA0x02 + seq_u16_le + PCM payload, current payload 180 bytes

              
END0x03

              
CANCEL0x04

              
STATUS0x10 + free + fill + received + read + high_water + active + finished

            

          


          

            
Streaming Flow

            
    
              
  1. Web server starts ffmpeg -i input -ac 1 -ar 16000 -f s16le pipe:1; ffmpeg continuously produces PCM.
    2. 
              
  2. The BLE sender is ffmpeg stdout's consumer: it reads PCM only when ESP32 credit says there is room.
    3. 
              
  3. Sender subscribes to Status Notify and treats it as batch ACK/credit: free, fill, received, read.
    4. 
              
  4. If credit is low, BLE sender stops consuming ffmpeg stdout; stdout blocks naturally, so ffmpeg is backpressured.
    5. 
              
  5. ESP32 BLE callback writes DATA into a PSRAM ring buffer and returns quickly.
    6. 
              
  6. ESP32 audio task starts after the low watermark, reads the ring buffer, expands mono to stereo, then writes I2S.
    7. 
              
  7. I2S DMA moves stereo PCM to ES8311 while the BLE producer continues only when credit allows.
    8. 
            

          


          

            
ESP32 Internal Pipeline

            
BLE GATT write callback
  -> validate seq
  -> PSRAM ring buffer producer
  -> Status Notify credit updates

audio task on CPU0
  -> wait low watermark
  -> ring buffer consumer
  -> mono s16le to stereo s16le
  -> i2s_channel_write()
  -> ESP-IDF I2S DMA
  -> ES8311 speaker output

          


          

            
Backpressure Contract

            
Sender:
  wait until free >= payload + safety_margin
  write DATA(seq)
  locally subtract credit

ESP32:
  notify on subscribe/start/finish/cancel
  notify after every 4 KB consumed
  report free/fill/received/read/high_water

          


          

            
Producer / Consumer Ownership

            
PC side:
  ffmpeg = PCM producer
  BLE sender = ffmpeg consumer + BLE producer
  ESP32 status notify = batch ACK / credit source

ESP32 side:
  BLE callback = ring-buffer producer
  audio task = ring-buffer consumer + I2S producer
  I2S DMA = hardware consumer to ES8311
